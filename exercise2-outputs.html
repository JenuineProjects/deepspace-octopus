2. Steelman Against Complexity

Core Thesis

"Configuration IS the distraction." Every settings screen consumes the exact cognitive resource (executive function) that the user is trying to protect. The app designed to save your focus destroys it before you begin.

Psychological Principles

PrincipleApplication to DeepSpace
Decision FatigueFocus app config depletes the same executive function pool needed for deep work. Zero-config preserves cognitive resources.
Paradox of ChoiceMore options = decision paralysis. One-click eliminates the paradox entirely.
Zeigarnik EffectIncomplete config creates persistent cognitive load. Zero config = zero Zeigarnik tension.
Implementation IntentionsAI-generated sprint tasks serve as automatic implementation intentions, releasing cognitive tension.
Cognitive Load TheoryWorking memory holds 4±1 chunks. Minimize extraneous load (config) to maximize germane load (deep work).
Flow State PreconditionsFlow requires clear goals. Config inserts meta-goals before real goals. One-click provides clarity.
Ego DepletionWillpower spent on config decisions depletes self-control needed for focus.

Key data point: It takes 23 minutes 15 seconds to refocus after an interruption. If 5 min of app config precedes a 25-min sprint, the user may not achieve full focus until minute 10 — losing 40% of productive time.

6. One-Click-Start Critical Path

1. App launches -> loads settings (or creates defaults)
2. Blocker checks for stale blocks from previous crash -> cleans up
3. UI renders: single "Start Focus" button + collapsed goal field
4. User clicks "Start Focus"
   -> Timer starts immediately (25 min default)
   -> Blocker activates with default blocklist
   -> AI planner runs async in background (if goal + API configured)
   -> Notification: "Focus session started"
5. User is focusing. Total elapsed time: < 1 second from click.

The timer never waits for AI. The planner enriches but never blocks.

7. Integrated Phase 1 Blueprint Recommendation

Two-Stage Build Approach

Stage 1 (MVP): Timer engine + distraction blocker + UI. One-click-start with zero AI dependency. Validate the core loop.

Stage 2 (Enhancement): Add AI sprint planner, session history, and progressive disclosure settings.

This mirrors the master intent: ship the simplest thing that delivers value, then layer intelligence on top.
